FIX Specification
 

We created a parser to go through the FIX specs and generated well formed XML versions. We use this XML version to generate most of the code for our FIX engine. Seeing as though it is so useful for us, we figured other people might be interested too.

This is a machine readable version of the FIX specification. If you have been looking for such a thing, this is it!

  • FIX 4.0 xml html
  • FIX 4.1 xml html
  • FIX 4.2 xml html
  • FIX 4.3 xml html
  • FIX 4.4 xml html

    • The XML documents were then reformatted into small browsable HTML files using this xsl stylesheet


    FPL Repository XML Conversion
     

    The FPL has recently released an xml repository for the FIX protocol. This repository is available to FPL members only. We have created a script that will convert from the FPL repository into a QuickFIX data dictionary in the format of the ones available above. Simply place this script in the root directory of the unzipped schema, and run it. It will scan for all directories in the format of FIX.x.y and create a FIXxy.xml file where x is the major and y is the minor version of FIX. You will need a ruby interpreter.

    Download Script: fix_to_quickfix.rb
