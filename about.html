About QuickFIX
 

QuickFIX is a full-featured open source FIX engine, currently compatible with the FIX 4.0-4.4 spec. It runs on Windows, Linux, Solaris, FreeBSD and Mac OS X. API's are available for C++, Java, .NET, and Python. Learn more.


About FIX
 

The Financial Information eXchange (FIX) Protocol is a "language" which defines specific kinds of electronic messages for communicating securities transactions between two parties. FIX defines only the format of the messages and the session-level interaction between two applications. More information on FIX can be found at the FIX protocol web site.


Scripting FIX
  June 29, 2004

Integration of FIX into a trading system is often viewed as a big complicated thing, and in large systems, it sometimes can be. But does it always have to be?

At it's core, FIX does something really simple. No matter how much someone may go on about "Enterprise Solutions", the fact is you just want to move data from one place to another. Sometimes you need all the bells and whistles required by a large-scale deployment, but sometimes you just need to shove some data down the pipe.

An interesting thing happened in the financial industry a couple of years ago. Traditionally based on C and C++, more firms began adopting Java as their primary development platform. Moving from C++ to Java is not interesting in itself. What is interesting is the idea of CTO's shifting focus from pure execution speed, and beginning to concentrate more on rapid development. They saw Java as a way to achieve this.

Scripting languages like Ruby and Python have proven themselves to be wonderfully robust rapid development environments. At the cost of execution speed, you can put together systems in days which could take weeks with higher maintenance languages like C++ or even Java. For processes that don't need to eek out every last CPU cycle, my hope and belief is that these languages will become adopted for more diverse tasks.

Although several nice libraries like quantlib have scripting interfaces through python, there are few options for connectivity. Well, thanks to the magic of SWIG, this is changing. As of version 1.8.0, QuickFIX comes with a full-featured Python API. Now you can take those "prototypes" you have implemented in python and connect them to real trading systems. And since the core is still implemented in C++, you still retain many of the speed advantages over a pure python implementation.

Next time someone dumps a list of trades in Random File Format v.136.98b that must be pushed over a FIX connection, you will have a proper tool to do this. You can rapidly implement your automated trading strategies and forward test them on a live connection. And heck, if it makes money, why re-implement it anyway.

Replacing complexity with simplicity is my main motivator right now. Less C++ and more scripting will make my life easier. I can write more functionality with less scaffolding. Spend time running code instead of compiling it. What can I say? The summer makes me lazy. I'd rather do more in less time with less effort. Isn't that why we invented computers to begin with?
